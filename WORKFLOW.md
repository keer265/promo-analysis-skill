> 约束：首次调用阶段只收集固定配置项，不额外追问分析类型、业务背景、行业；如有必要，也应放到 readiness gate 之后再补问。

# WORKFLOW

本 workflow 不是固定死板模板，而是以完成分析目的为主：在数据条件允许的范围内，自由组合时序分析、基线分析、增量评估、分层分析与运营策略推导。

默认分析主线：
- 识别活动与优惠投入如何影响销售表现
- 结合月度消费周期、周内周期、活动窗口解释时段差异
- 评估潜在效果与局限性
- 探索更合理的优惠策略与运营优化方向
- 输出可落地的用户运营方案

本文件定义 promo-analysis-skill 的标准分析工作流。

---

## 总体目标

从“表面增长”中识别“真实增量”，解释不同时间段销售变化的来源与差异，评估活动与优惠投入的潜在效果和局限性，并把分析转成可执行的优惠优化与用户运营方案。

## Phase -1. 调用配置（新增）

### 目标
在正式分析前，先确认运行条件与交付条件。

### 必做输出
1. 是否已提供数据路径/数据来源
2. 输出格式：Word / HTML / 两者都要
3. 输出路径
4. 若选 HTML，是否指定风格

### 原则
- 若未提供数据路径/数据来源，必须先询问
- 用户已提前给出时，不要重复追问
- 必须采用**一步一问**：一次只确认一个配置项
- 当前项确认后，再进入下一项
- 不要在一条消息里把全部配置项同时问完
- 该阶段完成前，不进入 readiness gate

## Phase 0. 阶段闸门

### 目标
根据当前已经读取到的数据，先判断“能执行到哪一块”。

### 必做输出
1. 已读取表与关键字段
2. 当前执行等级（Level 0~4）
3. 当前可执行模块
4. 当前受阻模块
5. 降级分析方案
6. 最后一问：继续 / 仅输出缺口 / 停止

### 原则
- 未完成该阶段前，不进入完整分析正文
- 若用户选择“继续”，再进入 Phase 1~10
- 若用户选择“仅输出缺口”，就在此结束

## Phase 1. 数据理解与检查

### 目标
判断数据是否足以支撑完整分析。

### 输出
- 数据表概览
- 字段完整性检查
- 主键与日期一致性检查
- 异常值与缺失值说明
- 结论：标准模式 / 最简模式 / 不足以分析

### 关键问题
- 是否具备订单表？
- 是否具备活动表？
- 是否具备曝光表？
- 日期能否统一到日级？
- 同日多活动如何处理？

---

## Phase 2. 分析口径统一

### 目标
统一业务口径，避免后续结论前后不一致。

### 需明确
- GMV 口径
- Pay UV / Purchase UV / Exposure UV 口径
- 优惠金额口径
- 活动日定义
- 活动窗口定义
- 类目与 SKU 粒度

### 输出
- 指标字典
- 口径说明
- 分析边界

---

## Phase 3. 核心底表构建

### 目标
将原始数据整理为支持不同分析视角的底表。

### 建议底表
1. `SKU-day`
2. `Category-day`
3. `User-day`
4. `Event-window`

### 输出
- 各底表粒度说明
- 样本量说明
- 关键衍生字段说明

---

## Phase 4. 描述性统计与周期识别

### 目标
先识别自然消费节奏，再谈活动效果。

### 至少完成
- 日级 GMV / 订单数 / 用户数时序
- 周内周期
- 月内周期
- 活动日 vs 非活动日对比
- 重点类目的周期差异

### 推荐图表
- 日级三指标时序图
- 月份 × 日期热力图
- ACF 图
- 月内周期曲线
- Top 类目热力图

### 输出
- 是否存在周内周期
- 是否存在月末高峰
- 哪些类目强周期，哪些弱周期

---

## Phase 5. 反事实基线构建

### 目标
估计“如果没有活动，本来会卖多少”。

### 推荐做法
- 用非活动日样本训练
- 排除活动前后 buffer
- 用固定效应 / 可解释加性模型
- 对 GMV 做适度变换以缓解偏态

### 最低要求
- 给出建模样本定义
- 给出模型思路
- 给出拟合优度或至少拟合表现说明
- 给出残差合理性描述

### 输出
- baseline 预测值
- 模型效果说明
- 限制说明

---

## Phase 6. 活动效果评估

### 目标
衡量活动带来的真实增量而不是表面销售额。

### 至少包括 4 项
1. 活动窗口增量评估
2. 事件研究分析
3. 增量来源拆解
4. 累计增量曲线

### 核心口径
- `incremental_gmv = actual_gmv - baseline_gmv`
- `incremental_rate = incremental_gmv / baseline_gmv`
- `iROI = incremental_gmv / discount_amount`

### 输出
- 各活动窗口实际 GMV / 基线 GMV / 增量 GMV
- 哪次活动最好 / 最差
- 增量主要来自什么
- 活动效应持续多久

---

## Phase 7. 因果验证与稳健性补充

### 目标
增强识别可信度。

### 可选方法
- 类目分组 DID
- 合成控制
- Oaxaca-Blinder 分解（描述性补充）
- 安慰剂检验
- 窗口稳健性

### 原则
- 不要过度宣称因果
- 若识别条件不足，要明确说是辅助验证

### 输出
- 方法选择原因
- 核心结果
- 与主线基线模型是否一致
- 不确定性说明

---

## Phase 8. 提前消费与蚕食效应识别

### 目标
判断活动增长中有多少不是“真正新增”。

### 提前消费
观察活动后窗口：
- 有多少样本低于基线
- 活动后平均增量如何衰减

### 蚕食效应
观察 SKU 层：
- 有多少共同 SKU 在活动期无正增长
- 哪些品牌 / 类目内部替代更明显

### 输出
- 活动是否存在回吐
- 回吐程度如何
- 哪些商品更可能被替代

---

## Phase 9. 用户分层与优惠敏感度

### 目标
把“谁对优惠敏感”讲清楚。

### 推荐做法
- 构建用户画像特征
- 规则分层
- 聚类验证
- 画像总结

### 重点维度
- 月底偏好
- 活动敏感度
- 消费额
- 客单价
- 折扣使用率
- 品类多样性
- 最近购买间隔

### 输出
- 核心用户群体
- 哪类用户值得投放
- 哪类用户不宜过补

---

## Phase 10. 商品敏感度与折扣响应

### 目标
识别哪些商品值得补贴、哪些不值得。

### 商品敏感度
用两个维度划分：
- 基础需求强弱
- 活动敏感度高低

### 折扣响应
观察不同折扣区间：
- GMV 变化
- 转化代理变化
- 是否存在边际收益递减

### 输出
- 四象限商品建议
- 最有效折扣带
- 不宜加深折扣的对象

---

## Phase 11. 预算优化模拟

### 目标
回答“预算应该怎么花”。

### 基础思路
把“活动 × 类目”或“活动 × 单元”作为优化粒度，计算：
- 增量 GMV
- 优惠投入
- iROI

### 推荐情景
- 当前全量投放
- 80%预算 + iROI排序
- 仅高 iROI 投放
- 低折扣深度策略

### 输出
- 各情景预算 / 增量 / iROI
- 是否存在边际收益递减
- 推荐优先投放单元

---

## Phase 12. 文件交付

### 目标
在完成分析后，形成最终交付。

### 输出形式
- Word 报告
- HTML 报告
- 或两者都给

### 原则
- 最终报告必须与 Phase 0 的执行等级一致
- 做不到的部分不能伪装成“已完成”
- 降级分析必须在报告里写明边界
- HTML 交付物必须是可编辑的独立文件
- HTML 文件应支持在浏览器中保存为 PDF

---

## Phase 13. 对话框检查输出（新增）

### 目标
在文件交付后，用对话框补充风险提示与人工复核建议。

### 必做输出
1. AI 幻觉/事实一致性检查
2. 证据不足项
3. 占位符未替换项
4. 过强推断风险
5. 需要人工复核的图表或数字

### 原则
- 这一部分**只输出到对话框**
- 不写入 Word 或 HTML 文件
- 不把不确定内容伪装成确定结论
