# RUN_SETUP

本文件定义 skill 在**刚被调用时**必须先确认的运行配置。

目标：
在正式读取和分析前，以**向导式分步确认**的方式，把“背景信息、数据从哪里来、文件往哪里出、输出要什么形式、还有哪些必要选项”逐项确认清楚。

关键要求：
- **一次只问一个配置项**
- 当前项确认后，才进入下一个配置项
- 不要把全部配置项同时展开
- 路径类用输入框样式
- 选项类用按钮样式
- 背景信息只收**简短输入**，不要扩展成泛化访谈

---

## 一、强制原则

1. **刚开始先收一句背景信息**
   - 这不是泛化追问，而是固定配置项
   - 用于记录活动背景、项目背景、特别约束或输出偏好
   - 只要一小段即可；用户也可填写“无”

2. **若未提供数据路径/数据来源，必须再问**
   - 不能直接进入阶段闸门
   - 不能假设数据位置
   - 不能默认当前对话一定已有可用数据

3. **若已提前给出数据路径/数据来源，就不要重复追问**
   - 用户已给文件路径、目录路径、已上传文件说明、多表映射时，直接复用

4. **输出格式与输出路径必须在前面确定**
   - 避免分析完才发现交付形式不对
   - 避免文件输出目录不明确

5. **必要选项也要显式选择**
   - 若输出 HTML，再确认风格预设

---

## 一点五、禁止提问项（非常重要）

首次调用时，**不要**额外追问以下内容：
- 你具体想分析什么
- 分析类型是什么
- 评估单个活动还是多个活动
- 请先展开讲业务背景
- 请先详细介绍行业

原因：这些问题不属于首轮运行配置，容易把固定配置页变成泛化咨询问答。

只有在以下情况才允许补问：
1. 用户主动提出多个互相冲突的分析方向；
2. 数据字段口径严重歧义，且不补问就无法进入 readiness gate；
3. 用户明确要求先讨论业务问题，而不是直接跑 skill。

默认做法是：**先收齐固定配置，再做闸门检查，再视需要补问极少量口径问题。**

## 二、首次调用的标准确认顺序（必须一步一问）

### Step 1. 背景信息（输入框）
先收一句背景信息，例如：
- 活动背景 / 项目背景
- 特殊输出要求
- 本次最关注的问题
- 特别约束（例如只看某类活动）

可接受形式：
- 一句话简述
- 两三句简短说明
- “无”

### Step 2. 数据路径/数据来源
若未提供，再问：
- 数据文件路径是什么？
- 或者数据所在文件夹路径是什么？
- 或者你要我使用当前上传的哪些文件？

可接受形式：
- 单个文件路径
- 文件夹路径
- 多个文件路径
- 已上传文件名 / 文件集合
- 表名与路径映射

### Step 3. 输出格式（必须三选一）
- **A. Word**
- **B. HTML**
- **C. 两者都要**

### Step 4. 说明两者区别（必须明确说明）
- **Word**：完整版商业分析报告，内容更完整，适合评审、归档、批注和继续修改；**正文中必须插入关键图表/图像，不能只有文字和表格**。
- **HTML**：精简版汇报材料，只放核心结果、核心数据、核心发现、核心图以及建议；页面内必须有可见图表或图像位。
- **HTML 的额外要求**：必须保持为可修改文件，修改后可以在浏览器中保存为 PDF；并支持替换页面中的图像内容。

### Step 5. 输出路径
必须确认：
- 生成的文件输出到哪个路径
- 若用户只给项目目录，也应明确输出子目录

### Step 6. 其他需要用户选择的事项
除了上面 5 项外，还要让用户选择以下内容：

1. **若选择 HTML，风格预设是什么**
   - Executive Blue
   - Strategy Gray
   - Growth Red

> 其余如报告对象、分析重点、输出粒度、字段映射等，不再作为首次调用的必问项。
> 默认由 skill 围绕分析目标自主展开；只有在字段歧义或业务口径冲突时，再补充询问。


## 三、推荐的首次调用弹窗式模板

若用户尚未给足信息，建议**直接用更像产品交互面板的写法**，而不是普通问答列表。
要求：
- **一步一屏 / 一次一问**
- 当前项确认后，才显示下一项
- 选项用“按钮感”写法呈现
- 路径项用“输入框/填写框”写法呈现
- 不加额外解释，不追加泛化追问
- 用户已提供的项标记为“已填写”，直接跳到下一缺失项

### 推荐模板（完整向导版）

#### Step 1：背景信息（输入框）
```text
【1/5 背景信息】
┌ 请输入本次项目/活动的简短背景信息（可填“无”） ┐
│                                                      │
└                                                      ┘
[ 确定 ]
```

#### Step 2：数据来源（输入框）
```text
【2/5 数据来源】
┌ 请输入文件路径 / 文件夹路径 / 已上传文件名 ┐
│                                                  │
└                                                  ┘
[ 确定 ]
```

#### Step 3：输出格式（按钮单选）
```text
【3/5 输出格式】
[ A Word ]
[ B HTML ]
[ C 两者都要 ]
```

#### Step 4：输出路径（输入框）
```text
【4/5 输出路径】
┌ 请输入导出目录 ┐
│                │
└                ┘
[ 确定 ]
```

#### Step 5：HTML 风格（仅 HTML 或两者都要时出现）
```text
【5/5 HTML 风格】
[ Executive Blue ]
[ Strategy Gray ]
[ Growth Red ]
```

#### 在 Step 3 选格式时，必须同时给出简短说明
```text
Word：完整版商业分析报告，适合评审、归档、批注和继续修改；文中必须插入关键图表/图像
HTML：精简版，只保留核心结果、核心数据、核心发现、核心图和建议；文件支持直接修改、替换图像，并可打印/保存为 PDF
```

### 推荐模板（缺项补问版）
当用户已经给出部分信息时，不要重列全部项目，只进入下一个缺失项。例如用户已经给了背景信息和数据来源，则直接从输出格式开始：

```text
【下一步：输出格式】
[ A Word ]
[ B HTML ]
[ C 两者都要 ]
```

### 强制规则
- **禁止整页罗列所有配置项**
- **禁止把多个按钮组同时展示**
- **禁止“一条消息里把所有配置都问完”**
- **必须模拟产品向导：点一个，确认一个，再进入下一个**

### 禁止写法
不要改写成：
- “请告诉我你的分析类型”
- “请简单描述业务背景”
- “请先介绍行业和目标”
- “由于这是一个专门用于……的技能，我需要先了解……”


## 四、默认值（仅在用户允许默认时使用）

- 背景信息：无
- HTML 风格：Executive Blue

---

## 五、调用配置完成后的收口

当上述信息确认完成后，再进入 readiness gate，并说明：

> 配置已确认：我将按你提供的背景信息、选择的数据来源、输出格式和输出路径继续；
> 下面先做可执行性判断，再决定正式分析范围。
