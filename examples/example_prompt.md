# example_prompt

下面是一个更像产品交互弹窗的推荐触发示例。

---

请使用 **promo-analysis-skill-v3**。

如果我没有给背景信息或数据路径/数据来源，请不要直接分析，也不要问分析类型；请严格采用**一步一问**的向导式提问，当前项确认后再进入下一项。例如先问：

```text
【1/5 背景信息】
┌ 请输入本次项目/活动的简短背景信息（可填“无”） ┐
│                                                      │
└                                                      ┘
[ 确定 ]
```

确认后再问：

```text
【2/5 数据来源】
┌ 请输入文件路径 / 文件夹路径 / 已上传文件名 ┐
│                                                  │
└                                                  ┘
[ 确定 ]
```

再确认后再问：

```text
【3/5 输出格式】
[ A Word ]
[ B HTML ]
[ C 两者都要 ]

Word：完整版商业分析报告，文中必须插入关键图表/图像
HTML：精简版，只保留核心结果、核心数据、核心发现、核心图和建议，且文件可修改、可替换图像，修改后可打印/保存为 PDF
```

然后依次再问：

```text
【4/5 输出路径】
┌ 请输入导出目录 ┐
│                │
└                ┘
[ 确定 ]
```

```text
【5/5 HTML 风格】
[ Executive Blue ]
[ Strategy Gray ]
[ Growth Red ]
```

配置确认后，再根据我上传的数据做 **readiness gate**：

1. 检查当前读到了哪些表和关键字段
2. 判断当前能执行到 Level 0 / 1 / 2 / 3 / 4 的哪一级
3. 列出能做、不能做、以及可以降级做的模块
4. 用一张简表告诉我：每个模块当前是否可执行
5. 最后询问我：
   - 继续执行当前可做部分
   - 只输出数据缺口与补充建议
   - 停止

如果我选择继续，再以完成分析目的为主，自由发挥，但至少优先覆盖：
- 统一 GMV、优惠金额、用户口径
- 识别月度消费周期、周内周期与活动窗口差异
- 构建反事实基线
- 评估活动与优惠投入的真实增量及其局限性
- 解释不同时间段销售变化的来源
- 分析是否存在提前消费、SKU 蚕食
- 做用户分层、商品敏感度和折扣响应分析
- 给出预算优化模拟
- 输出 Word 报告（正文记得插入关键图表/图像）
- 同时给一个精简版 HTML 汇报文件（页面内要有可见图）
- 最后把 AI 幻觉/事实一致性检查输出在对话框里

注意：首次调用阶段禁止追问“你想分析什么”“分析类型”“业务背景/行业”。
只有在完成上述配置并做完 readiness gate 后，如字段口径仍冲突，再补问最少量必要问题。


## 首轮配置展示风格

首轮配置优先使用按钮式选项与输入框式路径，但必须是**逐步出现**，不要整页展开。例如先只显示：

```text
【1/5 背景信息】
┌ 请输入本次项目/活动的简短背景信息（可填“无”） ┐
│                                                      │
└                                                      ┘
[ 确定 ]
```

等用户确认后，再显示：

```text
【2/5 输出格式】
[ A Word ]
[ B HTML ]
[ C 两者都要 ]
```
