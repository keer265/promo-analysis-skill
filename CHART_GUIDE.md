# CHART_GUIDE

本文件规定本 skill 里所有图表的解释规范。

---

## 总原则

**每张图都必须解决两个问题：**
1. 这张图在看什么？
2. 这张图说明了什么？

禁止只给图，不给解释。

---

## 每张图后至少包含两段说明

### 1) 读图指引
用于告诉读者怎么读图。

建议结构：
- 横轴是什么
- 纵轴是什么
- 颜色 / 分组代表什么
- 应重点关注哪一段 / 哪个拐点 / 哪组差异

### 2) 关键洞察
用于总结业务意义。

建议结构：
- 先说结论
- 再说证据
- 最后说含义或行动启示

---

## 推荐模板

### 读图指引模板
```text
读图指引：
本图横轴为____，纵轴为____。
不同颜色 / 线型分别表示____。
重点关注____到____阶段的变化，以及____与____之间的差异。
```

### 关键洞察模板
```text
关键洞察：
从图中可以看到，____。
这一结果说明____。
对业务而言，意味着____。
```

## 不同图的解释重点

## 1. 时序图
重点说：
- 是否存在峰谷
- 是否有趋势抬升
- 活动日附近是否有明显跳升
- 活动后是否回落

## 2. 热力图
重点说：
- 高值集中在哪些区域
- 是否存在月底 / 周末聚集
- 哪些类目模式明显不同

## 3. 基线拟合图
重点说：
- 基线是否贴合非活动日
- 活动日是否系统性高于基线
- 模型是否能刻画自然波动

## 4. 残差图 / QQ图
重点说：
- 是否以 0 为中心
- 是否存在明显偏态 / 厚尾
- 是否影响结论可靠性

## 5. 事件研究图
重点说：
- 活动前是否已预热
- 活动日及活动后是否保持抬升
- 是否呈现宽峰型而非单点爆发

## 6. 分解图
重点说：
- 哪个因素贡献最大
- 哪个因素形成拖累
- 活动是靠流量、效率还是客单拉动

## 7. 分层图 / 雷达图
重点说：
- 哪类用户 / 商品最突出
- 哪些维度差异最大
- 分层的业务含义是什么

## 8. 预算效率前沿图
重点说：
- 是否存在边际递减
- 哪个预算区间最有效
- 推荐方案为何不是全量投放

## 图表语气要求

### 要做
- 结论先行
- 语言简洁
- 说明“是什么 + 为什么重要”

### 不要做
- 把图上肉眼可见的信息机械复述一遍
- 使用过多统计术语堆砌
- 在证据不足时写成强因果结论

---

## 推荐句式

### 适合
- “结果显示……”
- “从图中可以看到……”
- “这说明……”
- “与……相比，……更明显”
- “意味着后续应优先……”

### 不适合
- “显而易见……”
- “充分证明……”
- “绝对说明……”
- “毫无疑问……”

---

## HTML 报告下的推荐说明框

可放在图后：

### 读图指引框
- 浅色背景
- 左侧边框
- 3~4 行即可

### 关键洞察框
- 深一点的左侧边框
- 2~3 个 bullet
- 强调行动含义


---

## 报告交付中的强制要求

### Word 报告
- 关键分析章节必须插入图表或图像
- 不要只给文字和表格
- 每张关键图后建议紧跟“读图指引 + 关键洞察”

### HTML 报告
- 页面内必须默认展示可见图表或图像位
- 图像应支持替换，替换后导出的 HTML 仍要保留图像
- 图后仍应保留简短说明框
